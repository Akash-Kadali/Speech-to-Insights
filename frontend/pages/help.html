Frequently Asked Questions

1. What kind of files can I upload?
You can upload .mp3, .wav, .m4a, .txt, or .pdf files via the Upload page.
2. How does semantic search work?
The search engine uses embedding similarity to match your natural-language query against transcripts. It retrieves the most relevant segments from your meetings.
3. What are “sessions”?
Sessions represent individual meetings or recordings you uploaded. Open any session to view transcripts, highlights, and insights.
4. Why am I seeing demo data?
This project includes an optional demo mode so instructors and graders can explore features without uploading files.
5. Where is my data stored?
For this MSML 650 demo, data is stored locally or simulated. This version does not use cloud storage or user authentication.

How to Use This System

Upload

Go to Upload to add a meeting. Fill the fields, pick a file, and submit. Demo mode seeds sample entries for testing.

Search

Ask natural questions like “What decisions were made about pricing?” Results show matching transcript snippets with context.

Sessions

View uploaded meetings, open a session to read the transcript, see highlights, and exported summaries.

Analytics

Dashboards include topic frequency, speaker participation, and keyword patterns across sessions.

Admin Panel

Admin simulates queue status and demo data for presentations. It's a UI demo, not a production control panel.
