Help & FAQ

Find answers and learn how to use each part of the system.

Frequently Asked Questions

1. What kind of files can I upload?
You can upload .mp3, .wav, .m4a, .txt, or .pdf files via the Upload page. (Accepted types may be restricted by the server in some deployments.)
2. How does semantic search work?
The search engine uses embedding similarity to match your natural-language query against transcripts. It retrieves the most relevant segments from your meetings and returns snippets with context.
3. What are sessions?
Sessions represent individual recordings or meetings you uploaded. Open a session to view the transcript, highlights, and exported summaries.
4. Why am I seeing demo data?
This project includes an optional demo mode so instructors and graders can explore features without uploading files. Demo data is safe and read-only in this environment.
5. Where is my data stored?
For the MSML 650 demo, data is stored locally or simulated. This version does not include user authentication or production cloud storage unless explicitly configured on the server.

How to Use This System

Upload

Go to Upload to add a meeting. Choose a file, fill the optional metadata, and submit. In demo mode you can seed sample entries for testing.

Search

Ask natural questions such as “What decisions were made about pricing?” Results show matching transcript snippets and links to sessions.

Sessions

View uploaded meetings, open a session to read the transcript, jump to timestamps, and export summaries.

Analytics

Dashboards show topic frequency, speaker participation and keyword patterns. If no backend analytics are configured, placeholder charts are shown.

Admin Panel

The Admin view simulates queue status and demo data for demos and grading. It's not a production control panel—treat it as a UI demonstration.

Still need help?

If the documentation doesn't answer your question, use the Contact page to send a message. This demo uses a mock contact form — no real emails are sent unless you wire a backend.
