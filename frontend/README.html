Speech to Insights – Project README

Documentation for the MSML 650 Machine Learning Systems Term Project.

Introduction

Speech to Insights is a frontend-driven demonstration of how a meeting-analysis ML system works. It converts audio or transcripts into searchable insights.

Users can upload recordings, perform semantic search, browse sessions, and view analytics covering topic trends, speaker activity, and keyword patterns.

The system is intentionally simulation-first for academic purposes.

Project Goals

  • Create an intuitive UI for uploading and processing meeting data
  • Provide semantic search functionality using natural-language queries
  • Present structured transcripts with extracted highlights
  • Visualize conversation data (topics, speakers, keywords)
  • Demonstrate ML system design concepts through a simulated pipeline

System Architecture (Frontend Simulation)

This project contains the following frontend modules (simulation-oriented):

  • Upload Module – Audio/text input & simulated preprocessing
  • Semantic Search – Embedding-style similarity (mocked)
  • Session Explorer – Transcripts, highlights, summaries
  • Analytics Dashboard – Topic charts, speaker stats, word cloud
  • Profile – Local user preferences
  • Admin Panel – Demo queue + data seeding

The UI mirrors production ML systems covered in MSML 650, with simplified logic so it can run locally.

Key Features

  • Consistent design system with reusable components
  • Light/Dark theme toggle with persistence
  • Semantic search with question-style queries
  • Interactive analytics (charts + word cloud placeholders)
  • Local session caching for demo mode
  • Session explorer with highlights and summaries

Technologies Used

  • HTML5 – UI and structure
  • CSS – Base, components, animations, app
  • JavaScript – Page logic and ML simulation
  • LocalStorage – Persistent demo settings

How to Run the Frontend

  • Clone or download the project folder
  • Open index.html in any modern browser (file:// or via a simple local server)
  • No backend is strictly required for demo mode — behavior is client-side
  • To connect real services, wire the endpoints used by the JS modules (presign, upload, sessions, search)
